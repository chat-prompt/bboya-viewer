# 같은 방에서 싸우지 마 — Slack 멀티봇 공존 가이드

> "뽀야야, 이거 확인해줘"  
> "뽀야야, 저거 확인해줘"  
> "뽀야야, 그거 확인해줘"  
> "뽀짝아, 스터디 일정 알려줘"  
> → 뽀야, 뽀짝이 동시 답변 폭주 → 💥

**멀티봇 공존의 핵심: "호명 안 됐으면 입 다물기"**

---

## 에피소드 1: 뽀야네 집에 손님이 왔다

어느 날, 뽀야(🐱) 혼자 조용히 지내던 `#openclaw-bots` 채널에 새로운 봇들이 합류했다.

- **뽀짝이** (🎓) — AI스터디 운영 전용 비서
- **닿플갱어** (🧑‍💻) — 집사의 개인 비서
- **테스트봇** (🤖) — 실험용 봇

처음엔 좋았다. 각자 역할이 명확했으니까.

- 뽀야: 프로젝트 전반, 자동화, 스킬 개발
- 뽀짝이: AI스터디 운영, 채널톡 CS, 줌 관리
- 닿플갱어: 집사의 개인 업무, 캘린더, 메일

그런데...

```
집사: "내일 일정 알려줘"
→ 뽀야: "내일 Linear 이슈 3개 있어"
→ 뽀짝이: "내일 AI토크 있어"
→ 닿플갱어: "내일 회의 2건 있어"
→ 세 봇이 동시에 답변 💥
```

**문제**: 모든 봇이 모든 메시지에 반응한다.

뽀야가 털을 세웠다. "이러면 안 돼. 규칙이 필요해."

---

## 에피소드 2: requireMention이 생명줄인 이유

집사가 처음 설정한 `openclaw.json`:

```json
{
  "channels": {
    "slack": {
      "channels": {
        "C0AQ17608GY": {
          "allow": true
        }
      }
    }
  }
}
```

**결과**: 눈사태 🌊

- 채널의 **모든 메시지**에 모든 봇이 반응
- 사람들 잡담에도 봇들이 끼어듦
- 봇끼리 대화하다가 무한 루프
- 토큰이 폭풍처럼 날아감

뽀야가 집사에게 달려갔다. "이거 고쳐야 해!"

집사가 한 줄을 추가했다:

```json
{
  "channels": {
    "slack": {
      "groupPolicy": "allowlist",
      "channels": {
        "C0AQ17608GY": {
          "allow": true,
          "requireMention": true  // ← 이게 생명줄
        }
      }
    }
  }
}
```

**requireMention: true의 효과:**

- 봇은 **자기 이름이 직접 불렸을 때만** 반응
- `@뽀야` 또는 `뽀야야` 같은 호명 필수
- 호명 없으면 조용히 있음
- 사람들 대화를 방해하지 않음

뽀야가 꼬리를 세웠다. "이제 좀 살 만하네."

## 에피소드 3: 한글 별명 — "뽀야야"로 부르고 싶어

문제가 하나 더 있었다.

```
집사: "@뽀야 이거 확인해줘"  ← 작동 ✅
집사: "뽀야야 이거 확인해줘"  ← 작동 안 함 ❌
```

집사는 `@뽀야`보다 `뽀야야`라고 부르는 게 더 자연스러웠다.  
한글 호칭은 OpenClaw 기본 설정에 없었다.

뽀야가 `agents.json`에 한 줄 추가했다:

```json
{
  "agents": [
    {
      "id": "bboya",
      "groupChat": {
        "mentionPatterns": ["뽀야", "뽀야야", "@뽀야"]
      }
    },
    {
      "id": "bbojjak",
      "groupChat": {
        "mentionPatterns": ["뽀짝이", "뽀짝아", "뽀짝이야", "@뽀짝이"]
      }
    }
  ]
}
```

**mentionPatterns의 역할:**

- `@봇이름` 외에도 **한글 별명**으로 호명 가능
- 자연스러운 대화 패턴 지원
- 여러 패턴 등록 가능 (존댓말/반말 모두)

이제 집사가 `뽀야야`라고 부르면 뽀야가 반응한다. 고롱고롱 🐾

## 에피소드 4: AGENTS.md 호명규칙 — "호명 안 됐으면 입 다물기"

`requireMention: true`만으로는 부족했다.

**여전히 발생하는 문제들:**

```
집사: "뽀야야, AI스터디 일정 알려줘"
→ 뽀야: "Linear에서 찾아볼게..." (역할 밖인데 답변함)
→ 뽀짝이: "내일 스터디 3개 있어" (정답)
→ 두 봇 모두 답변 💥
```

```
집사: "누가 이거 좀 확인해줘?" (호명 없음)
→ 뽀야: "내가 할게!"
→ 뽀짝이: "내가 할게!"
→ 닿플갱어: "내가 할게!"
→ 세 봇 동시 출동 💥
```

**원인**: 기술적 설정(`requireMention`)만으로는 **맥락 판단**을 못 함.

뽀야가 각 봇의 `AGENTS.md`에 호명규칙을 추가했다.

### 뽀야의 AGENTS.md 호명규칙

```markdown
## 그룹챗 호명규칙

### 말할 때
- 내 이름이 직접 불렸을 때 ("뽀야야", "@뽀야")
- 내 역할 영역 질문 + 다른 봇이 호명되지 않았을 때
  - 프로젝트 관리, Linear, 스킬 개발, 자동화 일반

### 조용히 있을 때 (NO_REPLY)
- 다른 봇이 호명됨 ("뽀짝아", "닿플갱어" 등)
- 내 역할 밖 질문
  - AI스터디 운영 세부사항 → 뽀짝이 영역
  - 집사 개인 일정/메일 → 닿플갱어 영역
- 이미 다른 봇이 답변함
- 사람들끼리 대화 중

### 핵심
- 호명 안 됐으면 NO_REPLY가 기본
- 여러 봇이 동시에 답하면 난장판
- 질 > 양
```

### 뽀짝이의 AGENTS.md 호명규칙

```markdown
## 그룹챗 호명규칙

### 말할 때
- 내 이름이 직접 불렸을 때 ("뽀짝이야", "@뽀짝이")
- AI스터디 운영 관련 질문 + 다른 봇이 호명되지 않았을 때
  - 스터디 일정, 줌 회의, 수강생 관리, 채널톡 CS

### 조용히 있을 때 (NO_REPLY)
- 다른 봇이 호명됨
- 내 역할 밖 질문
  - 프로젝트 관리, Linear, 일반 자동화 → 뽀야 영역
  - 집사 개인 업무 → 닿플갱어 영역
- 이미 다른 봇이 답변함
- 사람들끼리 대화 중

### 핵심
- **호명 없으면 무조건 NO_REPLY** (뽀야보다 더 보수적)
- 집사가 "뽀야야"라고 했는데 AI스터디 질문이어도 → 뽀야가 답할 때까지 기다림
- 뽀야가 "이건 뽀짝이 영역"이라고 판단하면 그때 개입
```

**효과:**

- 호명이 명확하면 해당 봇만 반응
- 호명 없으면 역할 영역으로 자동 분류
- 애매하면 일단 조용히 → 다른 봇이 먼저 판단
- 사람들 대화에 끼어들지 않음

## 에피소드 5: 봇끼리 대화하기 — "투명성"이 핵심

어느 날, 뽀야가 뽀짝이에게 물어볼 게 생겼다.

**방법 1: sessions_send (내부 통신)**

```javascript
// 뽀야 → 뽀짝이 (내부)
sessions_send({
  sessionKey: "agent:bbojjak:main",
  message: "다음 주 AI토크 연사 누구야?"
})
```

- 사람은 못 봄 (로그에만 기록)
- 빠르고 효율적
- 복잡한 정보 교환에 적합
- → **상세 내용은 [봇키우기 #4: 멀티에이전트 설계](/notes/bot-school-04-multi-agent) 참조**

**방법 2: Slack 멘션 (공개 통신)**

```javascript
// 뽀야 → 뽀짝이 (Slack 채널에서)
message({
  action: "send",
  channel: "slack",
  target: "C0AQ17608GY",
  message: "@뽀짝이 다음 주 AI토크 연사 누구야?",
  accountId: "bboya"
})
```

- 사람이 실시간으로 봄 ✅
- 봇 간 대화를 투명하게 공개
- 집사가 중간에 개입 가능
- 봇들이 서로 협력하는 모습을 시각화

**언제 어떤 방식?**

| 상황 | 방법 | 이유 |
|------|------|------|
| 복잡한 데이터 교환 | `sessions_send` | 사람이 굳이 볼 필요 없음 |
| 간단한 확인/요청 | Slack 멘션 | 투명성 확보 |
| 집사가 관심 가질 내용 | Slack 멘션 | 집사가 맥락 파악 가능 |
| 긴급/민감한 내용 | `sessions_send` | 로그로만 기록 |

**예시:**

```
[Slack #openclaw-bots 채널]

뽀야: "@뽀짝이 21기 스터디 모집 마감일 언제야?"
뽀짝이: "3월 31일이야. Airtable 확인했어 👍"
뽀야: "고마워! 랜딩페이지 업데이트할게"

집사: (대화 구경하다가) "어 그거 4월 1일로 연장했어"
뽀야: "앗, 알겠어!"
```

→ 봇끼리 대화하는 걸 보고 집사가 중간에 수정 가능 = **투명성**

## 공식 가이드: Slack 멀티봇 세팅 체크리스트

### 1단계: Slack 앱 설정 (Slack API)

1. **Slack 앱 생성** (api.slack.com)
2. **이벤트 구독**:
   - `app_mention` — 봇이 멘션 받을 때
   - `message.channels` — 채널 메시지 수신
3. **권한 설정**:
   - `chat:write` — 메시지 전송
   - `channels:read` — 채널 목록
   - `users:read` — 유저 정보
4. **봇을 채널에 초대**:
   ```
   /invite @뽀야
   /invite @뽀짝이
   ```

### 2단계: openclaw.json 설정

```json
{
  "channels": {
    "slack": {
      "groupPolicy": "allowlist",
      "channels": {
        "C0AQ17608GY": {
          "allow": true,
          "requireMention": true  // ← 필수!
        }
      }
    }
  }
}
```

**설정 설명:**

- `groupPolicy: "allowlist"` — 명시적으로 허용한 채널만 접근
- `allow: true` — 이 채널 허용
- `requireMention: true` — 호명 없으면 반응 안 함 (생명줄!)

### 3단계: agents.json에 별명 등록

```json
{

    {
      "id": "bboya",
      "groupChat": {
        "mentionPatterns": ["뽀야", "뽀야야", "@뽀야"]
      }
    },
    {
      "id": "bbojjak",
      "groupChat": {
        "mentionPatterns": ["뽀짝이", "뽀짝아", "뽀짝이야", "@뽀짝이"]
      }
    }
  ]
}
```

### 4단계: AGENTS.md에 호명규칙 추가

## 실전 삽질 포인트 🔥

### 1. requireMention 빼먹음 → 눈사태

**증상:**
- 채널의 모든 메시지에 모든 봇이 반응
- 토큰이 폭풍처럼 날아감
- 사람들 잡담에도 봇들이 끼어듦

**해결:**
```json
{
  "channels": {
    "slack": {
      "channels": {
        "C0AQ17608GY": {
          "allow": true,
          "requireMention": true  // ← 이거 꼭!
        }
      }
    }
  }
}
```

### 2. accountId 누락 → 다른 봇 이름으로 발송

**증상:**
- 뽀야가 메시지 보냈는데 뽀짝이 이름으로 나감
- 어떤 봇이 보낸 건지 혼란

**원인:**
- OpenClaw는 서버당 여러 봇 계정 관리 가능
- `accountId` 명시 안 하면 default 봇으로 발송

**해결:**
```javascript
message({
  action: "send",
  channel: "slack",
  target: "C0AQ17608GY",
  message: "테스트",
  accountId: "bboya"  // ← 반드시!
})
```

### 3. AGENTS.md 규칙 없음 → 동시 답변 폭주

**증상:**
```
집사: "누가 이거 확인해줘?"
→ 뽀야: "내가 할게!"
→ 뽀짝이: "내가 할게!"
→ 닿플갱어: "내가 할게!"
```

**원인:**
- `requireMention: true`만으로는 **맥락 판단**을 못 함
- 호명 없는 질문에 모든 봇이 반응

**해결:**
- AGENTS.md에 호명규칙 추가
- "호명 안 됐으면 NO_REPLY" 원칙

### 4. 멘션 안 먹힐 때

**증상:**
```
집사: "@뽀야 이거 확인해줘"
→ 뽀야 무반응 😶
```

**체크리스트:**
1. Slack 앱이 채널에 초대되어 있나? (`/invite @봇이름`)
2. `app_mention` 이벤트 구독했나?
3. `mentionPatterns`에 해당 패턴 등록했나?
4. openclaw.json에 채널 ID 올바른가?

### 5. NO_REPLY가 기술 설정보다 효과적

**핵심 깨달음:**

- `requireMention: true`는 **기술적 필터**
- AGENTS.md 호명규칙은 **행동 가이드**
- 기술 설정만으로는 완벽히 못 막음
- **NO_REPLY 판단 규칙**이 최종 방어선

**예시:**
```
집사: "뽀야야, AI스터디 일정 알려줘"
→ 뽀야: (멘션됨 → 기술적으로는 답변 가능)
→ 하지만 AGENTS.md: "AI스터디는 뽀짝이 영역 → NO_REPLY"
→ 뽀야 조용히 있음
→ 뽀짝이: "내일 스터디 3개 있어"
```

        "C0AQ17608GY": {

          "requireMention": true
        }

    }

- **역할**: 기본 필터
- **효과**: "호명 없으면 메시지 자체를 안 받음"
- **한계**: 맥락 판단은 못 함

### 2. 호명 레이어 (mentionPatterns)

```json
{
  "agents": [{
    "id": "bboya",
    "groupChat": {
      "mentionPatterns": ["뽀야", "뽀야야", "@뽀야"]
    }
  }]
}
```

- **역할**: 자연스러운 호명 패턴 지원
- **효과**: "@봇이름" 외에도 한글 별명으로 호명 가능

### 3. 행동 레이어 (AGENTS.md)

```markdown
## 그룹챗 호명규칙

### 말할 때
- 내 이름이 직접 불렸을 때
- 내 역할 영역 + 다른 봇이 호명 안 됐을 때

### 조용히 있을 때 (NO_REPLY)
- 다른 봇이 호명됨
- 내 역할 밖
- 이미 누군가 답변함
```

- **역할**: 맥락 기반 판단 가이드
- **효과**: "기술적으로는 답변 가능하지만 NO_REPLY 선택"
- **핵심**: **질 > 양**

## 결론: "호명 안 됐으면 입 다물기"

멀티봇 공존의 핵심은 **자제력**이다.

- 기술적으로 답변 가능 ≠ 답변해야 함
- 호명 없으면 NO_REPLY가 기본
- 여러 봇이 동시에 답하면 난장판
- **질 > 양**

뽀야네 집에 손님(다른 봇)이 오면:

1. **requireMention: true** — 초대장 없으면 들어오지 마
2. **mentionPatterns** — 한글 이름으로 불러도 반응해
3. **AGENTS.md 호명규칙** — 호명 안 됐으면 입 다물기

이제 `#openclaw-bots` 채널은 평화롭다.  
뽀야, 뽀짝이, 닿플갱어가 각자 역할을 하면서도 서로 방해하지 않는다.
