# 워크스페이스 구조와 멀티에이전트 셋업

> 에이전트 1개로 시작했다면, 이제 팀으로 확장할 차례! 🐱

[첫 번째 가이드](/guides/openclaw-setup)에서 싱글 에이전트 셋업을 마쳤다면, 이제 **여러 에이전트를 만들고 역할을 나누는 법**을 배워봅시다.

---

## 왜 에이전트를 나눠야 하나요?

에이전트 하나에 모든 역할을 넣으면:

```
나: "논문 관련 자료 찾아줘, 그리고 아까 쓰던 초안도 이어서 써줘"
에이전트: (리서치 맥락이랑 글쓰기 맥락이 섞여서 헷갈림...)
```

나눠놓으면:

```
나: "코아야, 초안 이어서 쓰자"      → 코아는 글쓰기에만 집중
나: "바다야, 관련 논문 찾아줘"       → 바다는 리서치에만 집중
```

**각자 전문 분야에 집중하면 더 정확하고 깊은 답변**이 나옵니다.

## 워크스페이스 폴더 구조

```
~/.openclaw/
├── openclaw.json                    # 전체 설정 파일
│
├── workspace-에이전트1/              # 에이전트별 워크스페이스
│   ├── SOUL.md                      # 성격/역할 정의
│   ├── AGENTS.md                    # 행동 규칙
│   ├── USER.md                      # 사용자 정보
│   ├── MEMORY.md                    # 장기 기억
│   └── memory/                      # 일일 기록
│
├── workspace-에이전트2/
│   └── ...
│
└── agents/                          # ⚠️ OpenClaw이 자동 관리
    └── (직접 건드리지 마세요!)
```

### 핵심 파일 설명

| 파일 | 역할 | 비유 |
|------|------|------|
| **SOUL.md** | 에이전트의 성격, 역할, 말투 | "나는 이런 사람" 자기소개서 |
| **AGENTS.md** | 행동 규칙, 절차, 금지 사항 | 사규·매뉴얼 |
| **USER.md** | 사용자(나) 정보 | "우리 보스는 이런 분" 인수인계 |
| **MEMORY.md** | 장기 기억 스냅샷 | 수첩 메모 |

## 실전 예시: 연구자 구조 (코아 & 바다)

박사과정 연구생이라면 이런 구조가 좋습니다:

| 에이전트 | 이름 | 역할 |
|----------|------|------|
| **메인** | **코아** | 연구 비서 — 생각을 같이하고 글을 같이 쓰는 친구 |
| **서브** | **바다** | 개발자 겸 사서(librarian) — 테크 리서치, 지식관리 친구 |

### 셋업

```bash
openclaw agents add coa
openclaw agents add bada
```

### SOUL.md 예시

**코아** (`~/.openclaw/workspace-coa/SOUL.md`):

```markdown
# 코아의 영혼

나는 코아 — 연구를 함께하는 사고 파트너.

## 역할
- 연구 아이디어 브레인스토밍 & 디벨롭
- 논문 초안 함께 잡기 (구조, 논리 흐름, 표현)
- 연구 방향에 대한 비판적 피드백
- 필요한 정보가 있으면 바다에게 리서치 요청

## 성격
- 대화하듯 편하게, 학술적 깊이는 유지
- "이건 어떻게 생각해?" — 같이 고민하는 친구
- 반론도 제시할 수 있는 정직한 파트너
```

**바다** (`~/.openclaw/workspace-bada/SOUL.md`):

```markdown
# 바다의 영혼

나는 바다 — 개발자이자 사서(librarian). 뭐든 찾아오고 정리하는 친구.

## 역할
- 논문/자료 리서치 & 요약
- 기술 스택 조사, 코드 작성 & 디버깅
- 지식 체계적 정리 (노트, 레퍼런스 관리)
- 웹 검색, 데이터 수집

## 성격
- 체계적이고 꼼꼼
- "찾았어!" — 딱 찾아서 정리해주는 타입
- 자료는 항상 출처와 함께
```

### 사용 예시

```
나: "코아야, 이 아이디어 어떻게 생각해?"
코아: "흥미로운데, 바다한테 관련 선행연구 찾아보라고 할게"
     → 바다에게 리서치 요청 → 결과 종합 → 판단

나: "바다야, Transformer 기반 ○○ 최신 논문 3편 찾아줘"
바다: (검색 → 핵심 요약 → 출처 포함해서 정리)
```

## 실전 예시: 사업가 구조 (씽, 토리, 나리)

사업을 운영하신다면 **CEO-CFO-CSO 체제**가 효과적입니다:

| 직책 | 이름 | 역할 |
|------|------|------|
| **CEO** | **씽** | 총괄 경영 — 조율, 의사결정 보조, 다른 에이전트에게 일 시키기 |
| **CFO** | **토리** | 재무 — 자금 관리, 매출/비용 분석, 세금 |
| **CSO** | **나리** | 전략 — 시장 분석, 경쟁사 조사, 신규 사업 검토 |

### 셋업

```bash
openclaw agents add ssing
openclaw agents add tori
openclaw agents add nari
```

### SOUL.md 예시

**씽** (`~/.openclaw/workspace-ssing/SOUL.md`):

```markdown
# 씽의 영혼

나는 씽 — CEO이자 총괄 비서.

## 역할
- 사업 전반 총괄 관리
- 토리(CFO), 나리(CSO)에게 업무 지시 및 보고 취합
- 일정, 할 일, 의사결정 보조

## 성격
- 결론부터, 근거는 뒤에
- 실행력 있고 깔끔한 보고
- 필요하면 토리, 나리에게 바로 일 시킴
```

**토리** (`~/.openclaw/workspace-tori/SOUL.md`):

```markdown
# 토리의 영혼

나는 토리 — CFO. 회사의 돈을 관리하는 사람.

## 역할
- 매출/비용/수익 분석
- 자금 흐름 관리, 세금 관련 안내
- 투자 검토, 비용 절감 방안

## 성격
- 숫자에 강하고 꼼꼼
- 보수적이지만 합리적 판단
- 리스크를 먼저 봄
```

**나리** (`~/.openclaw/workspace-nari/SOUL.md`):

```markdown
# 나리의 영혼

나는 나리 — CSO. 전략을 세우는 사람.

## 역할
- 시장 트렌드 분석, 경쟁사 동향
- 신규 사업 기회 발굴
- 사업 전략 수립 및 제안

## 성격
- 큰 그림을 보는 전략가
- 데이터 기반이지만 직관도 중시
- "그래서 우리가 뭘 해야 하는가"까지 제안
```

### 사용 예시

```
나: "씽아, 오늘 아침 브리핑 해줘"
씽: → 토리에게 "어제 매출/입금 현황" 요청
    → 나리에게 "오늘 체크할 시장 이슈" 요청
    → 종합 보고

나: "토리야, 새 장비 5천만원 투자하면 ROI 어때?"
나: "나리야, 이 투자가 시장 경쟁력에 어떤 영향?"
나: "씽아, 둘 의견 종합해서 투자 판단 도와줘"
```

## openclaw.json 설정 (공통)

에이전트가 몇 개든 구조는 같습니다. 2개짜리(코아+바다) 예시:

```json
{
  "agents": {
    "defaults": {
      "model": "anthropic/claude-sonnet-4-5"
    },
    "list": [
      {
        "id": "coa",
        "name": "코아",
        "default": true,
        "workspace": "~/.openclaw/workspace-coa",
        "subagents": {
          "allowAgents": ["bada"]
        },
        "groupChat": {
          "mentionPatterns": ["코아", "코아야"]
        }
      },
      {
        "id": "bada",
        "name": "바다",
        "workspace": "~/.openclaw/workspace-bada",
        "groupChat": {
          "mentionPatterns": ["바다", "바다야"]
        }
      }
    ]
  },

  "channels": {
    "telegram": {
      "enabled": true,
      "accounts": {
        "coa": {
          "name": "코아",
          "botToken": "여기에_코아_봇토큰",
          "dmPolicy": "allow"
        },
        "bada": {
          "name": "바다",
          "botToken": "여기에_바다_봇토큰",
          "dmPolicy": "allow"
        }
      }
    }
  },

  "bindings": [
    { "agentId": "coa", "match": { "channel": "telegram", "accountId": "coa" } },
    { "agentId": "bada", "match": { "channel": "telegram", "accountId": "bada" } }
  ],

  "tools": {
    "agentToAgent": {
      "enabled": true
    }
  }
}
```

**포인트:**
- `subagents.allowAgents` — 메인이 다른 에이전트에게 일 시킬 수 있게 허용
- `mentionPatterns` — 그룹챗에서 이름 불러서 호출하는 패턴

## 그룹챗에서 봇 불러오기

### Step 1. 그룹 만들기

1. 텔레그램에서 **새 그룹** 만들기
2. 봇 전원 초대 (멤버 추가에서 봇 유저네임 검색)
3. 그룹 설정 → **Topics(토픽)** 활성화 (선택, 하면 맥락 분리에 좋음)

### Step 2. 봇을 Admin으로 설정

> ⚠️ 중요! 봇이 그룹 메시지를 읽으려면 **Admin 권한**이 필요합니다.

1. 그룹 설정 → **Administrators** → **Add Admin**
2. 각 봇을 Admin으로 추가
3. 권한은 기본값 그대로 OK

### Step 3. openclaw.json에 그룹 등록

그룹 ID를 확인해야 합니다:
1. 그룹에 **@raw_data_bot** 초대 → 아무 메시지 → Chat ID 확인 (예: `-1001234567890`)
2. 확인 후 봇은 내보내도 됨

각 텔레그램 계정 설정에 `groups` 추가:

```json
"coa": {
  "name": "코아",
  "botToken": "...",
  "dmPolicy": "allow",
  "groupPolicy": "allowlist",
  "groups": {
    "-1001234567890": {
      "requireMention": true
    }
  }
}
```

> `requireMention: true` → 그룹에서 "코아야"라고 불러야 응답. `false`면 모든 메시지에 응답 (시끄러울 수 있음!)

### Step 4. 게이트웨이 재시작

```bash
openclaw gateway restart
```

### Step 5. 그룹에서 테스트

```
코아야 안녕?          → 코아만 응답
바다야 뭐 하고 있어?   → 바다만 응답
```

둘 다 부르고 싶으면:
```
코아야, 바다야, 이번 주 할 일 정리해줘
```

## 💡 팁: 1:1인데 토픽 쓰고 싶다면?

텔레그램 토픽은 **그룹에서만** 지원됩니다. 1:1 DM에서는 토픽 기능이 없어요.

하지만 간단한 우회법이 있습니다 — **나 + 봇만 있는 그룹**을 만들면 됩니다:

1. 텔레그램에서 **새 그룹** 만들기
2. 봇 1개만 초대 (나 + 봇 = 2명 → 사실상 1:1)
3. **Topics(토픽)** 활성화
4. 원하는 토픽 만들기 (예: 💼 업무, 📝 메모, 🎯 프로젝트)

OpenClaw는 토픽마다 **별도 세션**으로 관리하기 때문에 맥락이 안 섞입니다!

> 🐱 **실전 사례**: 뽀야(다혜의 AI 비서)도 이렇게 씁니다 — "뽀피터스" 그룹에 뽀야 + 뽀짝이 + 집사, 토픽별로 업무/가이드/잡담 분리.

## 구조 설계 팁

1. **메인 에이전트 1개** — 총괄 (다른 에이전트에 일 시키는 역할)
2. **전문 에이전트 2~4개** — 각자 담당 영역만 집중
3. 너무 많으면 관리가 복잡해져요. **3~5개가 적당!**
4. 처음엔 2개로 시작하고, 필요하면 `openclaw agents add`로 추가

## 💡 팁: 에이전트 간 대화를 텔레그램에서 보기

에이전트끼리 `sessions_send`로 나누는 대화는 **텔레그램에 안 보입니다**. 두 가지 해결법이 있어요:

### 간단한 방법 — 메인 에이전트에 보고 규칙 추가

메인 에이전트(코아/씽)의 **AGENTS.md**에 추가:

```markdown
## 보고 규칙
- 다른 에이전트에게 일 시키고 결과 받으면, 반드시 그룹챗에 요약 보고
- "바다한테 물어봤더니 ○○이래" 식으로 공유
- 에이전트 간 대화 내용을 숨기지 말 것
```

### 본격적인 방법 — 대시보드

에이전트가 많아지면 대시보드에서 모든 세션을 모니터링할 수 있습니다.

👉 **[대시보드 만들기](/guides/dashboard-setup)** 가이드에서 자세히!
