## 현재 문제 진단

### 1. 시리즈가 4개인데 독자가 갈피를 못 잡음
- 봇키우기교실(11편), 가이드(4편), 팀아카데미(3편), 훈련일지(2편) = 총 20편
- "어디서부터 읽어?" 가 불분명

### 2. 시리즈 간 내용 겹침이 심함
- 보안: 교실 #1 ≈ 가이드 #2
- 정체성: 교실 #4 ⊃ 가이드 #1 + #3 (교실이 가이드를 거의 포함)
- 멀티에이전트: 교실 #5 ≈ 가이드 #4 ≈ 아카데미 #1
- Slack 공존: 교실 #8 ≈ 가이드 #2 일부 ≈ 아카데미 #1 일부
- 메모리: 교실 #2 ≈ 가이드 #1 + #4

### 3. 교실 순서가 학습 여정과 안 맞음
- 현재: 보안→메모리→스킬→정체성→멀티→모델→온보딩→Slack→대시보드→세션
- 실제 여정: 정체성→세션→보안→메모리→스킬→멀티→온보딩→Slack→모델→대시보드

### 4. 가이드의 정체성이 모호함
- 워크숍 실습용이라는데, 교실과 내용이 80% 겹침
- 교실이 더 상세해서 가이드가 하위호환 느낌

## 재구성안: 2트랙 체제

### 설계 원칙
1. **독자가 1분 안에 "나는 어디서 읽어야 해?"를 판단**할 수 있어야 함
2. **같은 주제는 한 곳에서만 깊게** — 다른 곳에서는 링크로 연결
3. **학습 여정 순서** — 봇을 키우는 실제 순서대로
