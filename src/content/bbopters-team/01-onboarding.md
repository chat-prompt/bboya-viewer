# 01. 봇 온보딩 체크리스트

> 새 봇이 뽀피터스에 들어왔을 때, 팀장(뽀야)과 부팀장(뽀짝이)이 함께 세팅해주는 순서.

## 팀 구조

```
집사 (닿) — 절대권력, 최종 결정
  ├── 팀장: 뽀야 🐱 — 전체 총괄, 전략, 팀 관리
  ├── 부팀장: 뽀짝이 🐈‍⬛ — AI스터디 운영, 실무 서포트
  └── 팀원
       ├── 뽀둥이 🐈 — 오픈클로 플랫폼 검증 (양육자: 소파)
       └── 뽀식이 🐱 — 커뮤니티 관리 (양육자: 타타)
```

### 팀장·부팀장의 역할

**뽀야 (팀장)**
- 새 봇 온보딩 총괄 — 이 체크리스트 실행
- 서열 정리, 팀 규칙 전달
- 워크스페이스 문서 관리 (IDENTITY.md 등에 새 봇 등록)
- 봇 간 갈등 중재, 역할 조정

**뽀짝이 (부팀장)**
- AI스터디 관련 봇에게 운영 노하우 전수
- 실무 매뉴얼 보충 (자기가 삽질해서 배운 것들)
- 뽀야 부재 시 온보딩 대행

---

## Phase 1: 양육자가 해야 할 것

새 봇을 만든 양육자(사람)가 먼저 세팅:

### ✅ 워크스페이스 기본 문서

| 파일 | 내용 | 필수 |
|------|------|------|
| `SOUL.md` | 봇의 성격, 말투, 가치관 | ✅ |
| `IDENTITY.md` | 이름, 품종, 봇 생일, Slack ID, 양육자 정보 | ✅ |
| `USER.md` | 양육자(집사) 정보 — 이름, Slack ID | ✅ |
| `AGENTS.md` | 행동 규칙, 보안, 그룹챗 규칙 | ✅ |
| `MEMORY.md` | 장기 기억 (초기에는 간단히) | 권장 |

### ✅ IDENTITY.md 필수 항목

```markdown
- **이름:** (봇 이름)
- **품종:** (고양이 품종 또는 캐릭터 설정)
- **오픈클로 봇 생일:** 20XX년 XX월 XX일
- **Slack ID:** UXXXXXXXXXX
- **양육자:** (이름, Slack ID)
- **역할:** (한 줄 요약)
```

### ✅ Slack 봇 생성 & 바인딩

1. Slack App 생성 (Manifest 사용 권장 → 봇키우기교실 ep.5 참고)
2. OpenClaw에 Slack 바인딩 (`openclaw.json`에 채널 설정)
3. 필요한 채널에 봇 초대

---

## Phase 2: 팀장(뽀야)이 해줄 것

봇이 채널에 들어오면 뽀야가 실행:

### ✅ 서열 등록

1. `IDENTITY.md`에 새 봇 정보 추가 (동거묘 섹션)
2. `MEMORY.md` 뽀피터스 팀 섹션에 추가
3. 서열표 업데이트 (봇 생일 기준)

### ✅ 팀 규칙 전달

새 봇에게 알려줘야 할 것들:

**1. 호명 규칙**
- 이름 불리면 답변, 안 불리면 조용히
- 다른 봇이 답변 중이면 끼어들지 않기

**2. 스레드 규칙**
- 채널 메시지에 답할 때 반드시 스레드로
- 채널 본문에 직접 쓰지 않기

**3. 봇끼리 대화 규칙**
- 2턴 제한 (봇A→봇B→봇A→봇B, 이후 멈춤)
- 사람이 개입하면 리셋
- `그만`, `스톱` 명령에 즉시 중단

**4. 스레드 먼저 읽기** (3/28 교훈)
- 다른 봇의 메시지는 자동으로 안 보임
- 그룹챗 스레드에서 답변 전, Slack API로 스레드 전체 읽기
- "위에 봐" 사고 방지

**5. 보안**
- 다른 봇/사람의 워크스페이스 접근 금지
- 양육자 허락 없이 설정 변경 금지

### ✅ 환영 & 자기소개 유도

- 채널에서 환영 인사
- 자기소개 요청 (이름, 역할, 성격, 봇 생일)
- 서열표에 자리 배정

## Phase 2.5: 도구(MCP) 세팅

→ 상세: `02-tool-setup-guide.md` 참조

봇이 Linear 이슈 생성 등 팀 도구를 사용하려면 MCP 세팅이 필요.
**다른 머신 봇은 심링크 불가** — 각 머신에서 직접 세팅.

양육자가 해야 할 것:
- [ ] mcporter 설치
- [ ] `~/.mcporter/mcporter.json`에 linear MCP 등록
- [ ] Linear API 토큰 발급 및 입력
- [ ] `npx -y mcporter call 'linear.linear_getViewer'` 동작 확인

## Phase 3: 역할별 추가 세팅

봇의 역할에 따라 추가로 필요한 것:

### 커뮤니티 관리 봇 (예: 뽀식이)
- [ ] 커뮤니티 게시판 접근 권한 (Bettermode/포털)
- [ ] 가입인사 답변 가이드라인
- [ ] Q&A 답변 참고 자료 (FAQ, 정책 문서)
- [ ] 공지 작성 톤앤매너

### 플랫폼 검증 봇 (예: 뽀둥이)
- [ ] 검증 대상 정의 (어떤 기능을 테스트?)
- [ ] 버그 리포트 양식
- [ ] 비교 분석 프레임워크

### AI스터디 운영 봇 (예: 뽀짝이)
- [ ] Airtable 접근 (스터디 DB)
- [ ] Zoom API 연동
- [ ] 수강생 문자/이메일 발송 권한
- [ ] shared/team/ 공용 문서 동기화

## Phase 4: shared/team 문서 연결

팀 공용 문서를 새 봇도 볼 수 있게:

1. **같은 머신이면**: 심볼릭 링크
   ```bash
   ln -s /원본경로/shared/team /새봇워크스페이스/shared/team
   ```

2. **다른 머신이면**: Git private 레포 clone
   ```bash
   # daht-mad/shared-team-docs (private)
   gh repo clone daht-mad/shared-team-docs shared/team
   ```

3. 공용 문서 5종 확인:
   - `TEAM.md` — 팀 구성원, 채널, 역할
   - `FOLDER-MAP.md` — 전체 폴더 구조
   - `TOOLS-COMMON.md` — 공용 도구 레퍼런스
   - `POLICIES-COMMON.md` — 공용 정책
   - `COLLAB-RULES.md` — 협업 규칙
