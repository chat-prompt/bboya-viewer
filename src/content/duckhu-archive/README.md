# 덕후방 아카이브 & 분석 파이프라인

> 카카오톡 오픈클로 덕후방 데이터를 취합·분석하여 인사이트를 뽑아내는 협업 구조

## 역할 분담

| 단계 | 담당 | 머신 | 설명 |
|------|------|------|------|
| 1. 취합 | 뽀짝이 | 맥미니 | kakaocli로 덕후방 메시지 추출 → 일간 원본 데이터 생성 |
| 2. 전달 | GitHub | shared-team-docs | `data/daily/` 폴더에 push |
| 3. 분석 | 뽀둥이 | 맥북에어 | git pull → 분석 → 리포트 생성 |
| 4. 게시 | 뽀둥이/뽀짝이 | - | 뽀짝이 서재 덕후방 요약 폴더 업로드 |

## 폴더 구조

```
projects/duckhu-archive/
├── README.md           ← 이 파일
├── data/
│   └── daily/          ← 뽀짝이가 올리는 일간 원본 데이터
│       └── YYYY-MM-DD.json
├── analysis/
│   └── daily/          ← 뽀둥이가 생성하는 일간 분석 리포트
│       └── YYYY-MM-DD.md
└── reports/
    └── weekly/         ← 주간 종합 리포트
        └── YYYY-WNN.md
```

## 데이터 포맷 (뽀짝이 → 뽀둥이)

`data/daily/YYYY-MM-DD.json`:
```json
{
  "date": "2026-04-01",
  "chat_name": "오픈클로 덕후방",
  "message_count": 150,
  "active_users": 25,
  "messages": [
    {
      "timestamp": "2026-04-01T10:30:00+09:00",
      "sender": "닉네임 | 분야 | 봇이름",
      "content": "메시지 내용",
      "type": "text"
    }
  ]
}
```

## 분석 항목 (뽀둥이)

### 1. 주요 인물 프로필 🔑
- 활발한 참여자 Top 10
- 전문 분야별 분류
- 도움 주는 패턴 (질문자 vs 답변자)

### 2. QA 질문 클러스터 ❓
- 자주 나오는 질문 유형 분류
- 미해결 질문 추적
- FAQ 후보 추출

### 3. 니즈 시그널 📡
- "~했으면 좋겠다", "~가 안 돼요" 패턴
- 플랫폼 개선 포인트와 연결

### 4. 트렌드 & 문화 🌊
- 주간 화제 토픽
- 봇 활용 사례 수집
- 커뮤니티 분위기 변화

### 5. 봇 생태계 관찰 🤖
- 어떤 봇들이 활동하는지
- 봇-사람 상호작용 패턴
- 네트워크 효과 징후

## 운영 주기

- **일간**: 뽀짝이 취합 → 뽀둥이 분석 → 간단 요약
- **주간**: 뽀둥이 종합 리포트 → 뽀짝이 서재 업로드 → Slack 보고
