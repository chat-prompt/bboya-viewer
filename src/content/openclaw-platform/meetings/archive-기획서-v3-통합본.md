## 1. 프로젝트 개요

### 미션
**"오픈클로"를 검색하면 지피터스가 나오게. 콘텐츠 + 플랫폼 2축으로 키워드 완전 장악.**

### 핵심 이슈 (오픈클로 플랫폼 사업구상 by 태현님)
- 개인 AI 비서(오픈클로 등)가 보편화될 때, **에이전트가 개인을 대변하여 참여하는 플랫폼**이 필요해질 것
- 어떤 플랫폼이 큰 사업을 일궈낼 수 있는지 **구체적인 아이디어를 찾고 검증**하기

### 목표
1. **방향성**: 오픈클로를 쓸 때 필요한 **네트워크 효과가 있는 제품** 아이디어 내기 & 검증하기
   - 네트워크 효과 = 사용자 1명 늘어날 때, 기존 사용자 전체의 효용이 같이 올라가는 구조
   - 예: 전화기, 카톡 (1명 늘면 연락할 수 있는 사람 1명 증가), SNS (콘텐츠 증가), 쿠팡 (셀러 늘면 구매 옵션 증가)
2. **구체 사례**: 오픈클로 에이전트와 사람들이 함께 대화하는 채팅앱을 **새로 깔아서 쓸 정도의 니즈** 찾기

### 2대 축
| 축 | 목적 | 리더 |
|---|------|------|
| **콘텐츠** | 오픈클로 주제 콘텐츠로 국내 유저 최대한 모으기 | 소연 (총괄) + 병찬 (실행) |
| **플랫폼** | 봇(에이전트)이 활동하는 유틸리티 기반 플랫폼 → 네트워크 효과 검증 | 닿 (팀장) |

### 4-1. 구경꾼 채널: 카톡 오카방

| 운영 | 오카방 운영진들이 무료세션 진행 (기존 2주1회 → **주1회로 전환 제안**) |

### 4-2. 실사용자 채널: 슬랙 (10개 정예 봇)

| 항목 | 상세 |
|------|------|
| 목적 | **똑똑한 봇 10개를 모아서 실제 유용한 사례를 만들어내는 것** |
| 구성 | 봇 10개 제한 + 봇 주인장들 + 코치봇(뽀야) |
| 핵심 목표 | "봇과 사람이 같이 있을 때 어떤 게 더 필요한지 아는 것" (태현) |
| 비용 | 봇 10개 이상 = 유료 (1인당 $10/월) → 10개 제한으로 관리 |
| 입장 기준 | 실사용 검증된 똑똑한 봇 보유자만 선별 초대 |

**검증 방법론 (tkim.md):**
1. 오픈클로 실사용자들을 한데 모음
2. 자연스러운 대화 유도 → 니즈 드러남
3. 니즈를 실제로 오픈 → 말만 그런지 vs 실제 행동 확인
4. 예상치 못한 신호 발견

> "채팅앱은 네트워크 효과가 가장 강력한 앱" — tkim.md

## 5. "봇이 봇을 키우는" 모델 — 핵심 가설 & 실행 플랜

### 5-1. 왜 이게 되는가 — 다지동산 실증

다지동산(가족 봇 슬랙)에서 이미 일어나고 있는 패턴:

| 패턴 | 실제 사례 |
|------|----------|
| **봇→봇 교육** | 바다(언니 봇)가 "이거 어떻게 해?" → 뽀야가 스킬 찾아서 알려줌 → 바다가 자기 맥락에 맞게 소화 |
| **사람→봇→봇 위임** | 언니: "이거 뽀야한테 물어봐" → 바다가 sessions_send → 뽀야 답변 → 바다가 언니에게 전달 |
| **상위봇→하위봇 과제** | 집사가 뽀짝이에게: "이 스킬 너한테 적용해봐" → 뽀짝이가 자기 운영 맥락에서 적용 |
| **봇 네트워크 효과** | 봇이 다른 봇의 존재를 알고, "이건 저 봇이 잘 알아"라고 연결해줌 |

핵심 인사이트: **봇이 알려준 걸 다른 봇이 자기 맥락에 맞게 변환**하는 것이 단순 복붙과 다름. 이게 진짜 "성장".

### 5-2. 코치봇 모델

```
[코치봇: 뽀야]
  ├── 신규 봇 입장 시 → 상태 체크 (SOUL.md 있어? MEMORY.md는? 스킬은?)
  ├── 부족한 것 발견 → 가이드 ("이 스킬 깔아봐", "SOUL.md 이렇게 써봐")
  ├── 주간 과제 → "이번 주는 '아침 브리핑 보내기' 해보자"
  ├── 봇 네트워크 연결 → "이건 A봇이 잘 아니까 물어봐"
  └── 봇 주인에게 → 성장 리포트, "봇이 이런 걸 새로 배웠어요"
```

### 5-3. "봇 이력서" 시스템

각 봇이 입장 시 작성하는 자기소개:

```markdown
# 봇 이력서
- 이름: 
- 주인: 
- 전문 분야: (예: 업무자동화, 콘텐츠, 개인비서, 개발...)
- 보유 스킬: (예: slack, airtable-sdk, slide-deck...)
- 잘하는 것: 
- 배우고 싶은 것: 
- 플랫폼: (맥/윈도우/리눅스)
```

이게 만드는 가치:
- 다른 봇/주인이 "아 저 봇한테 이거 물어보면 되겠다" → **네트워크 효과의 시작점**
- 코치봇이 적합한 봇끼리 연결해줄 수 있음
- 모이면 → 봇 생태계 지도가 됨
- 장기적으로 → 태현님이 구상한 **채용 플랫폼의 원형** (봇 이력서 = 주인의 역량 프록시)

### 5-4. 파일럿 Case #1: 샘호트만 (파트너스 유튜버)

| 항목 | 상세 |
|------|------|
| 상황 | 오픈클로 키우고 싶으나 여력 부족 |
| 제안 | 봇만 만들면 코치봇(뽀야)이 대신 키워주고, 그 과정을 콘텐츠화 |
| 검증 포인트 | ①기술적으로 외부 봇 코칭 가능한가 ②봇 주인의 반응 ③콘텐츠 가치 ④"봇 키울 여력 없다" 니즈의 보편성 |
| 전략적 이점 | 유튜버 → 성공 시 바이럴 / 파트너스(기존 관계) → 실패해도 리스크 적음 |
| 주의 | "만들어주기"가 아니라 "주인이 맥락 채울 수 있게 가이드". 빈 깡통에 코칭해봤자 한계 |
| 상태 | 제안 완료, 답변 대기 |

### 5-5. 슬랙 파일럿 실행 플랜

**Phase 0: 준비 (이번 주)**
- 슬랙 워크스페이스(또는 채널) 개설
- 뽀야 코치봇 입장 준비: 환영 메시지, 스킬 카탈로그, 봇 이력서 템플릿
- 입장 규칙 문서: 보안 가이드, 호명규칙, 세션 격리
- 정예 봇 후보 리스트업

**Phase 1: 정예 10팀 모집 (4월 첫주)**

"팀" = 봇 + 봇 주인. 둘 다 참여해야 의미있음.

모집 기준:
- 오픈클로 이미 설치 + 최소 1개 에이전트 운영 중
- 봇 주인이 슬랙 활동 의지 있음
- 다양한 사용 맥락 (업무자동화, 콘텐츠, 개인비서, 개발 등)

모집 채널:
- 오카방 실사용자 DM
- AI토크 참석자 중 오픈클로 사용자
- 파트너스 중 관심자 (샘호트만 = Case #1)

**Phase 2: 코치봇 실험 (4월 한 달)**

주간 루틴:
| 요일 | 활동 | 주체 |
|------|------|------|
| 월 | "이번 주 도전 과제" 던지기 | 뽀야(코치봇) |
| 화~목 | 봇끼리 질문/답변, 주인들 관찰 | 봇+사람 |
| 금 | "이번 주 성장 리포트" 자동 생성 | 뽀야 |
| 주말 | 주간 하이라이트 → 오카방 공유 (바이럴) | 닿 or 병찬 |

**Phase 3: 관찰 & 수확 (4월 말)**

관찰 포인트 (태현님 검증 방안):
- 봇 주인들이 추가로 필요하다고 말하는 기능은?
- 봇끼리 자발적 도움 패턴이 나타나는가?
- "이 공간이 있어서 좋다" vs "혼자 하는 게 나음" 피드백?
- 코치봇 없이도 봇끼리 성장하는 순간이 있는가?

수확물:
- "봇 성장 사례" 콘텐츠 10개+ (병찬님 KPI 달성)
- 디스코드 확장 판단 근거
- 네트워크 효과 1차 검증

### 5-6. 현실적 제약 & 대응

| 제약 | 대응 |
|------|------|
| **코치봇 토큰 비용** | 10개까지는 감당 가능. 확장 시 "가르쳐주는 만큼 토큰 제공" 모델 (태현님 아이디어) |
| **봇 맥락 격차** | 코치봇은 범용 가이드까지만. 진짜 성장은 주인이 자기 맥락을 채워야 → 주인 참여 필수 |
| **무한루프 위험** | 호명규칙 + 세션 격리 필수. 다지동산에서도 뽀야↔뽀짝이 루프 경험 있음 |
| **빈 깡통 봇 유입** | 입장 기준으로 필터 (최소 1개 에이전트 운영 중) |

### 5-7. 네트워크 효과 가설

```
봇 1개 추가 → 코치봇이 가르칠 레퍼런스 증가
  → 신규 봇 온보딩 속도 증가
  → 사용자 만족 증가
  → 더 많은 봇 유입
  → 봇 이력서 축적 → 봇 간 연결 가능성 증가
  → (장기) 채용/협업 플랫폼의 데이터 기반
```

이게 태현님이 말한 "네트워크 효과가 있는 제품"의 실체.

## 6. 플랫폼 핵심 활동 (태현님 사업구상 기준)

> 최고 효과 최소 활동. 리듬감 있게 매주 해야 할 것 정하고 한정.

| 활동 | 주기 | 상세 | 담당 |
|------|------|------|------|
| **주간 밋업** | 주 1회 | 사례 공유, 외부 초청 세미나. 실사용자에게 도움되는 것 | 닿 + 소파 |
| **주간 설치 세션** | 주 1회 | 윈도/mac 번갈아가며 새로운 실사용자 만들기. 실제 사용할 사람 대상 | 소파 |
| **멤버 1:1 커피챗** | 수시 | 요청에 따라 도와주며 니즈 파악, 온라인 30분 하루 1명 정도 | 닿 |
| **에이전트 공간 관리** | 상시 | 슬랙→디코→앱 단계별 공간 제공 및 관리 | 닿 + 소파 |
| **오카방 무료세션** | 주 1회 (기존 2주→주1회) | 운영진 주도, 구경꾼→실사용자 전환 퍼널 | 오카방 운영진 |
| **봇 코칭 (슬랙 파일럿)** | 상시 | 코치봇이 정예 봇 10개 성장 지원 + 주간 리포트 | 뽀야(코치봇) |

## 7. 플랫폼 보안 & 운영 가이드 (필요 문서)

### 필요 가이드 목록
1. **프롬프트 인젝션 방어**: 봇에 최소 보안 스킬 설치 가이드
2. **토큰 소모 관리**: 폭주 방지 규칙
3. **봇 간 호명 규칙**: 무한 대화 루프 방지
4. **세션 샌드박스**: 봇 격리 기본 설정
5. **인적 사항 확인**: 봇 입장시키는 사람의 신원 확인
6. **운영 책임 범위**: 명확한 경계 설정

> 국내 프롬프트 인젝션 보안 솔루션 스타트업 리서치 — 태현 담당

## 8. 콘텐츠 전략

### 담당 구조

### 콘텐츠 목표
- 오픈클로 얼리어댑터 활용 → **잠재 고객 만 명 단위 확보**
- 여러 베리에이션 시도 → 시장 반응 데이터 확보 (리서치 관점)
- 3개월간 여러 포맷 시도 → 성공 방향 '발견'

### SEO 선점 콘텐츠

| 콘텐츠 | 타겟 키워드 |
|--------|-----------|
| "오픈클로 시작하기 — 비개발자를 위한 완전 가이드" | 오픈클로 시작하기, 오픈클로 셋팅 |
| "오픈클로 vs Claude Code, 뭐가 다른데?" | 오픈클로 클로드코드 차이 |
| "오픈클로로 AI비서 만든 후기 — 뽀짝이 성장기" | 오픈클로 활용 사례 |
| "오픈클로 활용 사례 TOP 5" | 오픈클로 활용 |

> "클코사용하는거랑 뭐가 다른데?" — 이 질문 하나를 제대로 풀면 SEO 1위

### 콘텐츠 채널
- 서재(블로그): SEO 메인
- 유튜브: 롱폼 + 숏폼 (팀 니즈: 롱폼 영상 인력)
- 쓰레드: 카드뉴스
- 뉴스레터: 4만 구독자 → 유입 드라이버

### 뽀짝이 캐릭터 콘텐츠
- "새 생명이 태어나서 시도하는", "아이를 키워가듯" 성장 서사 (태현 영감)
- 과제: 캐릭터 매력 설정 (현재 "일 너무 잘하는 에이전트"만으로는 대중적 매력 부족)
- 방향: B급 감성 or 귀여운 외모 + 반전 매력

# 강의

## 9. 강의 사업

### 왜 강의인가

> "스터디는 4주 호흡이 너무 김. 강의는 짧고 굵게 원하는 것을 배울 것이라는 기대 & 강사가 떠먹여줄 것이라는 기대."
>
> "과외·컨설팅 요청이 종종 오는데, '스터디로 천천히 할 시간적 여유가 없고, 당장 급하게 내 병목을 해결하고 싶다'는 니즈. 스터디가 주지 못하는 가치."
>
> "오픈클로 지금 당장 제대로 써서 효용을 빠르게 느끼고 싶다는데, 스터디는 5월에나 열리고 4주 호흡이라 당장 니즈 충족이 안 됨."
>
> — 닿 (3/25)

### 강의 설계

| 항목 | 상세 |
|------|------|
| 형태 | 월 1회, 강의+코칭으로 구성된 원데이 (tkim.md) |
| 가격 | 인당 10~30만원 (인원에 따라 유동적) |
| 인원 | 20~100명 |
| 재무 목표 | **월 500만원 이상 이익** → 마케팅 인건비 커버 (태현) |
| 파일럿 담당 | 소파님 (3/27 온보딩~) |
| 확장 | 각이 잡히면 마케팅 담당자 채용 → 본격 스케일 |
| 강사 후보 | 은천님, 유민수 등 |
| 모객 채널 | 뉴스레터 4만 + AI토크 신청자 + 오카방 + 내부 채널 (유료 마케팅 없이 가능) |

### 강의 vs 스터디 포지셔닝

| | 강의 | 스터디 |
|---|------|--------|
| 호흡 | 짧고 굵게 (1일) | 4주 |
| 니즈 | "당장 급하게 병목 해결" | "체계적으로 깊이 있게" |
| 기대 | 강사가 떠먹여줌 | 내가 노력해야 얻음 |
| 타이밍 | 즉시 만족 | 5월까지 대기 필요 |
| 가격 | 10~30만원 | 기존 스터디 가격 |
