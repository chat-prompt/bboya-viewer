## 0. 이슈

**지피터스 성장 전략과 Openclaw 기회의 우선순위가 명확하지 않음**

현재 상황:
- 지피터스는 기존 WAU 35K 목표를 달성해야 함
- Openclaw는 새로운 성장 기회이지만, 검증되지 않은 상태
- 두 목표를 동시에 추진할 경우:
  - 리소스 분산으로 둘 다 중반 포기할 수 있음
  - 콘텐츠 발행량/품질에 영향

## 1. 목표

### 1.1 결과물

**"지피터스 성장 + Openclaw 진입을 동시에 추진할 수 있는 통합 콘텐츠 전략 수립"**

- Openclaw 시장 수요 검증 및 성장 가능성 평가
- 두 목표를 리소스 낭비 없이 병행할 수 있는 콘텐츠 프레임워크

### 1.2 핵심 지표

- **26년 2Q 지피터스 평균 WAU 35,000**
  - 평균 WAU: **<25년 4Q>** 22,118 → **<26년 1Q>** 30,000 (+36.5%)
- Openclaw 관련 콘텐츠의 검색 유입 수 / 커뮤니티 활성화 정도

---

## 2. 리스크와 가설

### 리스크 1: 두 목표 동시 진행으로 기존 목표(WAU 35K) 달성 실패

발행 리듬이 깨지거나, 콘텐츠 품질이 떨어져서 기존 성장 모멘텀을 잃을 수 있음.

**→ (가설 1) 기존 발행 리듬 안에서 일부 콘텐츠를 Openclaw 테마로 "전환"하고, 추가 콘텐츠는 점진적으로 추가하면 기존 목표를 유지하면서도 Openclaw에 진입할 수 있다**

```
검증 방법:
주 2-3개 Openclaw 콘텐츠(기존 8개 중 전환) + 신규 튜토리얼 1-2개 추가
4월 말까지 지피터스 WAU 악화 여부 모니터링
```

### 리스크 2: Openclaw 검색 수요가 국내에서 충분하지 않을 수 있음

국내 콘텐츠는 설치 가이드 중심이고, 깊이 있는 가이드가 없는 상태. 실제 검색 수요가 있는지는 불명확.

**→ (가설 2) 검색 수요가 검증되지 않았다면, 우선 "설치→교육→커뮤니티" 퍼널에 집중하고, 검색 유입은 4월부터 모니터링한 후 피벗할 수 있다**

```
검증 방법:
3월: Openclaw 관련 콘텐츠 발행 시작
4월: Google Search Console에서 Openclaw 키워드 추적 시작
5월: 검색 유입 데이터 분석 후 전략 조정 결정
```

## 3. 가설 검증 계획

### 기존 실행 전략 (26년 1Q)

| 전략 | 내용 | 현재 발행 리듬 |
|------|------|--------------|
| SEO 최적화 | 검색 수요 기반 콘텐츠 | SEO 최적화 기반 게시글 8개 |
| SNS 확장 | 스레드, 숏폼, 카드뉴스 | 스레드 7-9개/주 |
| 뉴스레터 강화 | 월 1회 발행 + 포맷 점검 | 뉴스레터 1~2회/주 |
| 주간 리뷰 | 채널별 성과 분석 | 주 1회 |

### 3.1 기존 콘텐츠에 Openclaw 추가

**OpenClaw 콘텐츠 타겟:**
- OpenClaw에 관심은 있지만 아직 진입하지 못한 사용자층
- 진입 장벽 높음 → 교육/가이드 필요

**예상 퍼널:**
```
Openclaw 검색
    ↓
지피터스 발견 (검색엔진 or SNS)
    ↓
Openclaw 설치
    ↓
커뮤니티 흡수
    ↓
Openclaw 노하우 공유
    ↓
스터디 구매
```

- 주 2-3개의 게시글을 Openclaw 키워드로 작성
  - OpenClaw 튜토리얼 (온보딩)
  - OpenClaw 실전 활용 사례 중심
- 기존 발행 수 유지 혹은 추가 (주 8개 게시글 + 스레드 7-9개 + 뉴스레터 1~2회)

### 3.2 Openclaw 검색 수요 리서치 & 키워드 전략 수립

- Google Trends, Naver 검색량 분석
- Openclaw 관련 타겟 키워드 30개 이상 정리
  - 에이전트, AI 비서 등 연관 키워드 발굴 필요 (리터러시 높은 타겟용)
- 기존 국내 Claw 콘텐츠(설치 가이드)와의 차별화 포인트 정의

### 3.3 신규 Openclaw 콘텐츠 전략

- **온보딩**: 타겟에 맞는 OpenClaw 온보딩 콘텐츠 발행
  - 주 2~3개의 오픈클로 튜토리얼 (글/영상)
  - 오픈클로 웨비나를 유튜브에 업로드, 숏폼으로 재제작
  - OpenClaw 카톡방/게시판 기반 FAQ 콘텐츠화
- **실용성**: 국내에 없는 OpenClaw 활용 사례 발굴
  - 카톡방/게시판 활성화 → 사례 발굴 → 콘텐츠화
  - 뽀짝이가 실제로 어떻게 일하는지 유튜브 영상
- **간증**: 뽀짝이의 임팩트를 브랜드화/콘텐츠화
  - 타겟은 뽀짝이가 쓰는 글보다 뽀짝이의 존재/임팩트가 더 궁금할 것 (검증 필요)
  - 뽀짝이 스레드 → OpenClaw 커뮤니티 대표 캐릭터 활용 (like 콕집)
  - "신입 개발자 뽀짝이" 시리즈 숏츠 > 뽀짝이가 설명하는 튜토리얼
